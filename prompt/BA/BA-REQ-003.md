# Use Case Document

## Objective
Generate Use Case documentation for the feature described in the user input below.

## For Each Use Case Include

1. **Use Case ID & Name** — e.g. UC-001: Login
2. **Actors** — primary and secondary
3. **Preconditions** — what must be true before the use case starts
4. **Main Success Scenario** — numbered step-by-step flow
5. **Alternative Flows** — variations from the main flow
6. **Exception Flows** — error and failure scenarios
7. **Postconditions** — system state after successful completion

## Instructions
- Use active voice in step descriptions
- One action per step
- Reference other use cases where applicable
