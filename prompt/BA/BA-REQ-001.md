# Business Requirements Document

## Objective
Generate a Business Requirements Document (BRD) for the project described in the user input below.

## Required Sections

1. **Executive Summary**
   - Brief overview of the project and its business value

2. **Business Context**
   - Background, problem statement, and motivation

3. **Stakeholder Analysis**
   - List of stakeholders, their roles, and interests

4. **Functional Requirements**
   - What the system must do (numbered, clear, testable)

5. **Non-Functional Requirements**
   - Performance, security, availability, scalability

6. **Business Rules**
   - Constraints and policies the system must enforce

7. **Acceptance Criteria**
   - Conditions that must be met for delivery to be accepted

## Instructions
- Use clear, professional language
- Each requirement must be specific and testable
- Number all requirements (e.g. FR-001, NFR-001)
